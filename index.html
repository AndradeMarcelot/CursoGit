Estudos de Git

Esse documento foi elaborado com base em tutoriais e o guia rápido que segue: git - guia prático

O git é um versionador, isto é, um sistema com a finalidade de gerenciar diferentes versões de um documento. Para realizar o o registro adequado e seguro das versões ele trabalha com três árvores de diretórios:

  1. Work Directory - diretório criado para armazenar os arquivos do seu trabalho
  2. Index (staged) - estágio intermediário onde são colocados os arquivos em sua versão fechada, com a intensão de submetê-los para a terceira árvore
  3. HEAD - árvore com registro de todas as versões submetidas

Alguns comandos

git status

Exibe um relatório com os status dos arquivos no diretório, ajuda a identificar quais arquivos estão prontos para serem submetidos (commit).

git add * / git add nome_aquivo

Leva o arquivo do diretório de trabalho para a árvore index (staged). Nesse ponto os arquivos estão aguardando para serem submetidos (commit) e levados para a árvore HEAD, onde o pacote de modificações (versão fechada dos aquivos) são registradas no diretório local, mas com as modificações ainda não enviadas para o repositório remoto.
